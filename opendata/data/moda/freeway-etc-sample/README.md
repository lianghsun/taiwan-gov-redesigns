# 高速公路 ETC 通行記錄樣本

> ⚠️ **本資料完全為虛構示意**。
> 真實資料須向高速公路局與數位發展部申請。本平台僅作為「主權資料應該如何呈現」的 demo。

## 為什麼有這份資料

ETC 通行原始紀錄是高度敏感的個人位置資料；但聚合後（每 5 分鐘×門架×車種）能用於：

- 都市計畫之車流預測
- 交通管理（事件偵測、變動車道調控）
- 假期返鄉時段預測

因此主權資料的設計是：**以 5 分鐘窗口聚合、移除可識別資訊、發行樣本檔**。

## 欄位重點

- `gantryId` 命名規則：`N{國道}-{方向}-{里程km}`，例如 `N1-S-053` 為國 1 南下 53 公里處
- `vehicleClass` 採英文代碼，方便 SQL 與 ML 管線使用
- `count` 為窗口內通行次數；`avgSpeedKph` 為加權平均車速
