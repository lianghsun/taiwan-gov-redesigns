# 全民健保就醫紀錄差分隱私樣本

> ⚠️ **本資料完全為虛構示意**。
> 真實之主權資料須向數位發展部申請存取，本平台無法重現任何真實病歷。資料僅作為「主權資料應該長什麼樣」的 demo。

## 背景

主權資料的設計目標是讓研究者得以使用政府高敏感資料而不致洩漏個人隱私。常見處理手法：

1. 去識別化（Tokenization）
2. 分層彙總（Stratified bucketing）
3. **差分隱私（Differential Privacy）** — 在統計查詢中加入受控雜訊

本示意資料模擬第三種：每筆紀錄的欄位均已分桶，並對最終統計值添加 ε=1.0 的 Laplace 雜訊。

## Schema 重點

- `ageBucket`：避免精確年齡可被識別
- `feeBucket`：費用以區間表示
- `icd10Chapter`：僅揭露 ICD-10 章節層級，不揭露細項代碼

## 同樣的資料 demo 不能做的事

- 個案追蹤（沒有可追蹤的識別碼）
- 罕見疾病分布觀察（章節層級無法捕捉）
- 醫療機構排名（無院所欄位）
