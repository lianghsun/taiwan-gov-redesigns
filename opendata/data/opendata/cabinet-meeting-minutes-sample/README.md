# 行政院會議紀錄摘要（精選）

> **示範資料**：本 demo 收錄連續 3 期院會的 markdown 摘要。內容根據實際公告紀錄改寫，但**部分數字與發言為示意**，請以行政院官網公告為準。

## 為什麼以 markdown 提供

行政院會議紀錄原以 PDF 公告，雖然有「議事錄」標準格式，但 PDF 對於：

- 全文檢索 / 向量檢索
- 引用與超連結
- 變更追蹤（同一段落改了什麼）

都不友善。本平台示範把同樣內容改成：

- 一次院會 = 一份 `.md`
- H2 標示主要章節（議程、討論、決議）
- 表決結果用 task list 呈現
- 政策名詞於首次出現時加 inline link（demo 內為示意，未實際指向外站）

## 檔案命名

`YYYY-MM-DD-第N次院會.md`，可同時依日期與會次排序。

## 與結構化資料的差別

雖然紀錄裡確實有可表格化的資訊（表決票數、預算數字），但**整份文件的價值在於敘述脈絡**——誰提出、誰反對、為什麼。把它強行 schema 化反而會丟失這些資訊。
